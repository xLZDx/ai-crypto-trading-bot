## P0 — bot is down (must fix first)

### bug: 19 CRITICAL `could not convert string to float: ''` errors per symbol

**Root cause:** `src/utils/resample_ohlcv.py` writes one CSV row for every period in the date range. When the source 1s archive has a gap (e.g. exchange downtime, listing-date pre-history), `pd.resample` produces a row with NaN OHLC. Pandas `to_csv` writes NaN as an empty string (`,,,,,`), and the bot's `float(row['open'])` blows up.

Confirmed: BTC_USDT_1h.csv.gz has 128/68652 rows with empty OHLC cells, first at 2017-09-06 17:00 (the 1s archive has a 9-hour gap there).

**Fix:**
1. `_write_csv_gz`: `df.dropna(subset=['open','high','low','close'], inplace=True)` before writing.
2. Re-run resample for every existing CSV (fast — re-reads same chunks, just drops NaN rows).
3. The bot's CSV reader (`feature_reader._csv_load`) already wraps `float()` in `try/except` per-row — so even if a stale gap-row sneaks in later, the bot won't crash. But emit a one-time WARN if any are skipped so we notice.

**Verification:** `python -c "import gzip; ..."` per file — zero rows with `,,` between commas 2-5.
