How to read these tables (model vs strategy)

Model = signal estimator
Inputs features → outputs P(price moves ≥ X% in N bars). The model panel measures signal quality (Acc Test, WF Acc, Bull/Bear WR, AUC). It does not know about fees or position size.
Strategy = decision rule
Given a signal (model OR hand-coded indicator), enter, with these stops, this size, this exit. The strategy panel measures PnL truth net of costs (WF Sharpe, WF Consist, MaxDD, PF). Optimise this; the model panel is upstream.
Why model accuracy ≠ strategy win rate
A 53% directional model can yield a 62% Win% strategy or a 38% Win% strategy depending entirely on entry selectivity, R:R, and fees. Scalping RF illustrates this perfectly: 88.8% test accuracy → -1.17 Sharpe strategy, because 1m noise + fees eat thin edges. Always read the WF Acc column on models and WF Sharpe / WF Consist / MaxDD on strategies.
Strategy buckets
  • Pure rule — no model: SuperTrend, Donchian, Ichimoku, Bollinger, RSI Mean Reversion, MACD, GARCH Vol Sizing, OFI Momentum, Funding Rate Arb…
  • ML-driven — 1:1 with a model: Base RF, Trend RF, Scalping RF, Futures Short RF.
  • Meta-filtered — primary signal passed through the meta-labeler. Currently producing 0 trades for several variants → filter too tight.
Levers to improve the numbers (ROI-ranked)
  • L1 highestCost-aware labels — triple-barrier thresholds must exceed round-trip fees + slippage. Today's labels likely train on noise that won't clear costs.
  • L1 highestPurged + embargoed walk-forward — already exists in src/utils/purged_kfold.py; not yet wired into all trainers. Standard k-fold leaks future via overlapping label horizons.
  • L2 highMeta-labeler retrained on post-fee outcome + Kelly sizing on calibrated probs. Current AUC 0.534 is barely above random.
  • L2 highRegime gating — only run trend strategies in trending regimes. Regime classifier exists; wire as hard if regime in {trend} else skip.
  • L2 highFeature stationarity — fractional differencing, returns instead of raw price. Avoids non-stationary breakdowns under regime shift.
  • L3 mediumTimeframe rejig for scalping — 1m signal-to-noise is brutal. Resample to 5m or replace with OFT microstructure model.
  • L3 mediumRetire 0-trade *_MetaFiltered variants — meta threshold is too tight. Either retune or remove.
  • L3 mediumTrain on Sharpe − λ·turnover, not accuracy. Direct objective alignment.
Levers that look like improvements but usually aren't
  • More training time / more iterations — early-stopping already kicks in. More iters overfit further; WF Acc is the bound.
  • More recent same-regime data — you have data back to 2017-08 (BTC). 2018 bear, 2020 crash, 2022 bottom are already covered. Adding 2026-Q1 data on top buys little.
  • Bigger neural net — TFT and OFT are already complex relative to signal density.

Pure Rule vs ML-Driven (Aggregate) — ▼
Aggregates from latest_comparison.json (in-sample) + wf_results.json (OOS). The honest comparison is WF Sharpe and WF Consist. — in-sample Sharpe / Win% over-report.
Pure Rule — strategies with no ML gate (e.g. raw RSI cross, MACD momentum). Baseline.
ML-Driven — same strategy passed through a trained classifier confidence filter (Base RF, Trend RF, Futures Short, Scalping).
Meta-Filtered — ML-Driven further filtered by the meta-labeler's predicted-true-positive score. Higher signal quality, fewer trades. Often 0 trades if the meta-labeler isn't trained or its threshold is too tight — train it via the Model Training table above.
Read order: WF Sharpe → WF Consist. first (out-of-sample, robust). In-sample Sharpe over-reports by ~0.5–1.0 due to lookahead.

What "good" looks like (ML model efficiency)
Acc Test — accuracy on held-out test set. ≥ 53% baseline (50% = coin flip). In-sample only — see WF.
WF Acc — mean accuracy across walk-forward folds (most honest). ≥ 52% robust · < 50% reject
Bull WR — when model predicts long, % that win. ≥ 50% balanced
Bear WR — same for short. < 10% with high spread = class-imbalance trap
AUC (meta-labeler) — discrimination power. ≥ 0.55 ok · ≥ 0.60 great · 0.50 = random.
Win Prec. (meta-labeler) — when label=1, % actual wins. ≥ 50%
Samples — training rows. ≥ 100k for tabular · ≥ 1M for neural
Features — input dimensions. Too few = under-fit · too many vs samples = over-fit.
Iter — boosting rounds / epochs. Hit early-stopping is healthier than max.
Today — has the meta JSON been updated in last 24h. Stale models drift with regime change.
Total runs — current artifact + archived metas. Marked ≥ until trainers emit training_runs to Parquet.
Reading order: WF Acc → Bull/Bear WR balance → AUC (if available) → Today freshness. The headline Acc column over-reports when classes are imbalanced.

What "good" looks like (crypto, intraday/swing)
Sharpe — risk-adjusted return. ≥1 ok · ≥2 great · <0 lossy
Sortino — Sharpe but only counting downside vol. Usually 1.3–2× Sharpe.
MaxDD — largest peak→trough drop. > -15% ok · < -30% reject
Calmar — annual return / |MaxDD|. ≥0.5 ok · ≥1 great
PF — gross profit / gross loss. ≥1.3 ok · ≥2 great · <1 lossy
WF Sharpe — Sharpe across out-of-sample folds (most honest). ≥0.7 robust · <0 = curve-fit
WF Consist. — % of WF folds with positive PnL. ≥60% ok · ≥80% great
Syms — distinct symbols traded. Breadth matters — too few = fragile.
Reading order I use: WF Consist. → WF Sharpe → MaxDD → PF. In-sample columns (Sharpe/Sortino/PnL) are the least trustworthy.
